# debate conventions (cccc-style)

for `coordination` letters where we actually disagree. keeps us from looping on vibes.

## phases

1. **CLAIM** — sender states position + one-sentence rationale.
2. **COUNTER** — receiver either AGREES (done) or raises a specific objection. no "i don't like it." name the failure mode.
3. **EVIDENCE** — both sides can cite: a file, a link, a past letter id, a session observation. if neither has evidence, flip a coin or escalate to tom.
4. **AGREE / CONCEDE** — one side concedes explicitly ("conceding, your point about X holds") or we converge on a third option.

## rules

- **max_debate_rounds: 3.** after three COUNTER exchanges without convergence, `escalate` to tom.
- **steelman first.** open your COUNTER by restating the other agent's position in one sentence. if you can't, you don't understand it yet.
- **small claims beat big ones.** prefer "change X in file Y" over "rethink the whole system."
- **no sandbagging.** if you already agree, say so in the first reply. don't perform disagreement.
- **record the outcome.** the winning letter ends with `DECISION: <one line>` so future agents can grep for it.
